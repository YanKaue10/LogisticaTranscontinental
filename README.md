O Desafio da Malha Logística Transcontinental
A gigante global de logística, OmniRoute, opera uma frota de contêineres inteligentes monitorados em tempo real durante a travessia entre portos internacionais. Você foi designado para desenvolver o motor de análise de dados do terminal de cargas, que recebe um fluxo bruto de informações de sensores via Java. O conjunto de dados atual contém o identificador do contêiner, o peso da carga em toneladas, a temperatura interna em graus Celsius, a umidade relativa e o tempo de permanência no pátio em dias. O conselho administrativo exige que o sistema não apenas processe esses dados, mas gere uma planilha .csv de auditoria que classifique automaticamente cada unidade conforme as normas de segurança portuária.
Para que o relatório seja aceito pelo sistema alfandegário, o programa precisa realizar uma triagem rigorosa baseada em cruzamentos complexos: se um contêiner tiver peso superior a 25 toneladas e temperatura acima de 30°C, ele deve ser marcado como ALTO RISCO ESTRUTURAL, mas se o peso for menor que 5 toneladas com umidade acima de 80% por mais de 10 dias, o status deve ser RISCO DE AVARIA POR BOLOR. Além disso, o sistema deve aplicar uma regra de eficiência onde cargas que ocupam o pátio por mais de 30 dias com umidade instável (fora da faixa de 40% a 60%) recebam um alerta de REMANEJAMENTO PRIORITÁRIO. O resultado final deve ser exportado em uma planilha onde, além dos dados originais, conste uma coluna extra calculando o custo de armazenamento projetado, que é de R$50,00 por dia, mas sofre um acréscimo de 15% se o status for de risco. Como você estruturaria essa solução em Java para que o gestor portuário receba um arquivo pronto para ser aberto no Excel com todas as lixeiras e contêineres devidamente auditados, as médias de umidade por categoria de risco calculadas e a lista de unidades que devem ser movidas imediatamente?
